# sched/deadline: Fix schedstats vs deadline servers

- **Commit:** 9c602adb799e72ee537c0c7ca7e828c3fe2acad6
- **Affected file(s):** kernel/sched/deadline.c
- **Subsystem:** Deadline

## Bug Description

When schedstats is enabled and deadline servers are used, calling `dl_server_start()` triggers a kernel `BUG_ON()` assertion crash. The bug occurs because the scheduler tries to access task statistics on a deadline server entity (an internal scheduling construct) via `dl_task_of()`, but deadline server entities are not real tasks and do not have task statistics.

## Root Cause

The `__schedstats_from_dl_se()` function was missing a check to distinguish between real tasks and deadline server entities. When `dl_server_start()` sets `dl_se->dl_server = 1` and then calls `enqueue_dl_entity()`, the statistics update functions would eventually call `__schedstats_from_dl_se()` followed by `dl_task_of()`, which contains a `BUG_ON(dl_server(dl_se))` assertion that fails for deadline servers.

## Fix Summary

The fix adds two early-return checks in `__schedstats_from_dl_se()`: first to return NULL if schedstats is disabled, and second to return NULL if the entity is a deadline server. This prevents invalid `dl_task_of()` calls on non-task entities. The calling functions are updated to check if stats is NULL before proceeding with statistics updates.
